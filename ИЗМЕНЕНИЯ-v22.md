# Изменения — v22

## Вид производства с учётом звена суда (район ≠ область)

Дела «12-…» (жалобы на постановления по делам об АП, ст. 30.1 КоАП) в «Моих
делах» ошибочно показывались как гражданские: вид производства определялся
ТОЛЬКО по префиксу номера, и «12-» не попадал ни под одну ветку, проваливаясь
в «гражданское» по умолчанию. Причина глубже — один и тот же префикс на разных
звеньях суда значит разное: «2-…» на районном суде — гражданское (1-я
инстанция), а на суде субъекта — уголовное. По одному номеру, без звена, их не
развести.

Звено суда у каждого дела уже известно и хранится (`MovementContext.courtLevel`,
проставляется при импорте/поиске), а звено-зависимый классификатор номера уже
есть (`CartotekaRegistry.matches(caseNumber:level:)`). Теперь эта связка
задействована для определения вида производства.

### Классификация — `Sources/SudrfApp/AppModel.swift`

- `ProductionType.init(cartotekaId:)` — категория по ключу картотеки:
  `u*` → уголовное, `p*` → КАС, `adm*` → КоАП, `g*`/`m`/прочее → гражданское.
- `ProductionType.of(_:level:)` — при известном звене номер разбирается по
  картотекам этого звена (`CartotekaRegistry.matches`) → «12-…» → КоАП,
  «2-…» суда субъекта → уголовное. Без звена — фолбэк по номеру через
  канонический `ProcessKind.detect` (устранён дубль логики разбора).
- `TrackedCase` получил поле `production`, вычисляемое ОДИН раз при сборке
  строки в `makeTrackedCase` по цепочке: точная картотека из контекста
  (разрешена при импорте) → разбор с учётом звена → фолбэк по номеру.
  Читатели (живой фильтр таблицы, счётчики сайдбара, группировка «По
  производствам», строка списка) берут готовое `production` вместо пересчёта.
  Звено НЕ выводится из домена: эвристика `MovementService.courtLevel(forDomain:)`
  по умолчанию даёт `.subject` и спутала бы районные дела с делами субъекта.

### Канонический классификатор номера — `Sources/SudrfKit/Parties.swift`

- `ProcessKind.detect`: добавлен КАС-префикс «8а-»/«8a-» (кассация КАС в КСОЮ) —
  иначе фолбэк потерял бы прежнее «8а-» → КАС.
- Исправлен «7у-»/«7y-»: раньше относился к КоАП, хотя по реестру картотек это
  уголовная кассация КСОЮ (`u3: 77/7у`) — теперь уголовное.
