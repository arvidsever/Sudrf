# Изменения — Alpha 0.36.0

## CAPTCHA Assist без видимой страницы суда

Лист CAPTCHA Assist переведён в компактный нативный режим: пользователь видит
только крупную картинку, поле «Код», кнопку «Найти», статус и очередь дел этого
же суда. Полная страница суда остаётся в скрытом `WKWebView`, где сохраняются
cookies, hidden-поля и обычная отправка формы. Код по-прежнему читает и вводит
человек; OCR, Vision/Core ML и автоматическое распознавание не добавлялись.

Скрытый webview подставляет только обычные параметры поиска — УИД и номер дела,
извлекает CAPTCHA-картинку из той же страницы как `data:image/png`, затем после
ручного ввода записывает код в поле формы и отправляет настоящий поиск суда.

## Ретрай после успешного ввода

После успешной отправки сохраняется пара `captcha/captchaid` вместе с cookies, лист
закрывается, а приложение автоматически повторяет ожидающие поиски и обновления
для того же суда. Для карточек движения очищается старый cache entry, чтобы после
принятого кода сразу загрузилась свежая карточка, а не старая CAPTCHA-заглушка.

## Исправление КСОЮ

На сайтах КСОЮ успешная страница может содержать счётчик `counter.sudrf.ru`, в
URL которого закодирован предыдущий адрес с `captcha=...&captchaid=...`. Раньше
это принималось за новую CAPTCHA, из-за чего лист показывал ошибку «Не нашёл поле
кода» и мог оставлять старую картинку в панели. Детектор теперь ищет настоящую
форму CAPTCHA: видимое редактируемое поле `captcha` или подпись «Проверочный код /
код с картинки» рядом с полем ввода. Счётчики и закодированные ref/pg-параметры
не считаются CAPTCHA.

## Тесты

Добавлены тесты `CaptchaAssistTests` для решения post-submit state machine и
декодирования `data:`-картинок, а также регрессионные проверки КСОЮ в
`SearchPageClassifierTests`: inline CAPTCHA-форма распознаётся как CAPTCHA, но
успешная/пустая страница со счётчиком и закодированными `captcha/captchaid` — нет.

Полный прогон `swift test --quiet`: 219 тестов, 0 падений.
