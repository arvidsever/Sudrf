# Изменения — Alpha 0.29.0

## Уголовные дела: распознавание участников (вкладки «ЛИЦА» + «СТОРОНЫ»)

По четырём webarchive пользователя (Ленинский р/с Уфы, Сыктывкарский ГС) уголовные
карточки показывали «стороны не опубликованы», хотя участники на сайте есть.
Причина — в разборе карточки (`name_op=case`).

### Что было не так

На современном движке sudrf участники **уголовного (УПК)** дела лежат в **двух
отдельных вкладках/таблицах**:

- **«ЛИЦА»** — подсудимые: `Фамилия / наименование | Перечень статей | Дата |
  Результат`. Здесь колонка 0 — это ИМЯ, роль в вёрстке не указана.
- **«СТОРОНЫ»** — защитники, прокурор, представители потерпевших:
  `Вид лица | ФИО` (обычная раскладка «роль | имя»).

Старый `partiesTable` искал ЕДИНСТВЕННУЮ таблицу по тексту `<th>` (только
«СТОРОНЫ ПО ДЕЛУ» / «ЛИЦА, УЧАСТВУЮЩИЕ»). Реальные заголовки уголовной карточки —
«ЛИЦА» и «СТОРОНЫ» — не совпадали → `CaseParties` пустой → «стороны не
опубликованы». Заодно не распознавался заголовок «УЧАСТНИКИ» (гражданские
апелляция/кассация).

### Как исправлено

`CaseCardParser` (`parseParties`) переписан: тип таблицы участников определяется по
**заголовкам колонок**, а не по тексту вкладки `<th>` (он варьируется —
«СТОРОНЫ», «ЛИЦА», «СТОРОНЫ ПО ДЕЛУ», «СТОРОНЫ ПО ДЕЛУ (ТРЕТЬИ ЛИЦА)»,
«УЧАСТНИКИ»). Обходятся ВСЕ таблицы карточки:

- есть колонка **«Вид лица…»** → таблица сторон «роль | имя» (защитник, прокурор,
  представитель, а для КоАП — «ПРИВЛЕКАЕМОЕ ЛИЦО»);
- иначе есть колонка **«Перечень статей»** → таблица ЛИЦ: имя (колонка 0) +
  статьи; синтезируется под-роль «Подсудимый · ст.158 ч.3 п.г УК РФ».

**Порядок проверок критичен:** у КоАП таблица «СТОРОНЫ ПО ДЕЛУ» содержит ОБЕ
колонки — «Вид лица» И «Перечень статей», поэтому «Вид лица» проверяется первым
(иначе привлекаемое лицо КоАП уехало бы в разбор ЛИЦ и стало бы «Подсудимым»).
Таблица ЛИЦ со статьями встречается только в УПК.

Дальше уже существующая механика `CaseParties`/`buildSided` разводит участников по
«Стороне защиты» (подсудимый, защитник, привлекаемое лицо) и «Стороне обвинения»
(прокурор, потерпевший).

### Побочный фикс: карточка участников для УПК/КоАП

`CaseParties.isEmpty` опирался только на корзины (истец/ответчик/третьи) и
`columns`, игнорируя `roleItems`. Для УПК/КоАП, где стороны собираются из
`roleItems`, `isEmpty` ложно возвращал `true`, и `CaseMovementView` мог скрыть
карточку участников (гейт `!parties.isEmpty`). Теперь `isEmpty` опирается на
`displayColumns` — единый источник отрисовки.
