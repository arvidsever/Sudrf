# Изменения — v31

## «Списком»: перечисление нескольких сторон

В таблице «Списком» («Мои дела») при нескольких сторонах показывалась только
первая (например, дело 2-276/2026: 148 истцов, 10 ответчиков → по одному).

### Истцы/ответчики (ГПК/КАС)

Строка через «⚔» теперь перечисляет каждую сторону:

- один → «X»;
- двое → «X и Y»;
- трое и больше → «X и N других» (N = число − 1).

Склонение «другой/других» — через `DateUtil.plural`. Правка в
`MovementDerivation.partiesShort` (хелпер `namesShort`).

### Подсудимые/привлекаемые (УПК/КоАП)

В ячейке — несколькими строками:

- один → «ФИО ⟨щит⟩ статьи» (как прежде);
- двое → две строки, каждая «ФИО ⟨щит⟩ его статьи»;
- трое и больше → первая строка «ФИО ⟨щит⟩ статьи», вторая — «и N других»
  (N = число − 1).

Реализация: `CaseParties.chargedMembers` (участники со статьями),
`MovementDerivation.partiesSecondLine` → новое опциональное поле
`CaseSnapshot.secondPartyLine` (`PartiesSecondLine`) → `TrackedCase`. Отрисовка —
хелпер `chargedLine` в `MyCasesView`, применяемый к обеим строкам.

## Что не изменилось

Карточка дела и карточка выдачи (свои раскладки с колонками) — без изменений.

## Совместимость

Новое поле `secondPartyLine` — опциональное; старый кэш (`CaseSnapshot`)
декодируется без изменений, пересчитается при следующем обновлении движения.
