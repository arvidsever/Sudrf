# Что изменилось (v6 — исправление: GUID ссылки выдавался за УИД)

## Симптом

В шапке движения дела показывался не УИД, а посторонний идентификатор:
«УИД 0cec7ea2-1eae-47eb-988b-5df03f4f190c», хотя настоящий УИД дела —
`11RS0001-01-2025-011255-03`. Заодно молча не подтягивалась апелляция
(блока ВС Коми в движении не было вовсе), а окно капчи 3 КСОЮ подставляло
в поле «УИД» тот же GUID — поиск там тоже ничего бы не нашёл.

## Причина

В деле живут **два разных идентификатора**, и они перепутались:

| | Откуда берётся | Вид | Зачем нужен |
|---|---|---|---|
| `case_uid` (GUID) | параметр ссылки на карточку в выдаче (`case_uid=…`) | `0cec7ea2-1eae-…` | только для URL карточки; **у каждого суда свой** |
| УИД | строка «Уникальный идентификатор дела» в карточке | `11RS0001-01-2025-011255-03` | сквозной идентификатор дела по всем инстанциям |

Проверено на реальных карточках этого дела: 1-я инстанция (СГС), апелляция
(ВС Коми) и кассация (3 КСОЮ) имеют **разные** `case_uid`
(`0cec7ea2…` / `2aa596f3…` / `f2e11fa0…`) и **один** УИД. Поэтому поиск в
вышестоящем суде по GUID первой инстанции в принципе не мог ничего найти —
а `MovementService` отправлял в поле `…JUDICIAL_UIDSS` именно его, и его же
записывал в `CaseMovement.uid` (откуда он попадал в шапку и в окно капчи).

Парсер карточки при этом УИД извлекал правильно (`CaseCard.uid`) — значение
просто не использовалось.

## Исправлено

- `MovementService.movement`: УИД берётся из карточки 1-й инстанции
  (`baseCard.uid`); по нему идёт поиск в вышестоящих судах и он же кладётся в
  `CaseMovement.uid`. Если УИД в карточке не указан (старые дела), вышестоящие
  инстанции не перебираются впустую.
- `MovementService.minimalMovement`: больше не выдаёт GUID ссылки за УИД
  (УИД без карточки неизвестен — пустая строка).
- `SearchModel.runSearch`: УИД стал самым приоритетным полем сетевого запроса
  (он селективнее № дела). Удалён локальный дофильтр по УИД — он сравнивал
  введённый УИД с `caseUID` (GUID) и при комбинации «УИД + № / ФИО» всегда
  обнулял выдачу. Локальный дофильтр по № дела теперь по префиксу, потому что
  в выдаче номер идёт с дописками («2-7212/2025 ~ М-5922/2025»).
- `CaseMovementView`: если УИД неизвестен, в шапке честное «УИД в карточке не
  указан…» вместо пустого места.

Окно капчи 3 КСОЮ автоматически получает правильный УИД: оно берёт
`movement.uid`, который теперь настоящий.
