# Изменения — v21

## Импорт дел из CSV (выгрузка LegalHelp) + материалы в карточке дела

Перенос отслеживаемых дел из стороннего сервиса (lk.legalhelp.pw) в «Мои
дела»: webarchive со списком дел конвертируется скриптом в CSV, приложение
импортирует CSV через системное меню macOS. Попутно карточка дела получила
секцию «Материалы» (13-…, 3/12-…, 15-…) — раньше материалы в движении дела
не показывались вовсе.

### Скрипт выгрузки — `Scripts/export_cases_csv.py`

- `python3 Scripts/export_cases_csv.py "…/Список дел.webarchive" [выход.csv]` —
  разбирает webarchive (binary plist → HTML списка дел LegalHelp), достаёт
  по каждой карточке номер, суд, вид производства, стороны, дату обновления
  и прямую ссылку на карточку sud_delo.
- CSV в UTF-8 с BOM (Numbers/Excel сразу видят кириллицу), колонки:
  `number,court,kind,level,parties,updated,url`. Для импорта обязательны
  `court` и `url` — номер добывается из самой карточки.

### Импорт в приложении — Файл → «Импортировать дела из CSV…» (⇧⌘I)

- Новая команда в системном меню (первая команда приложения; сцена команд
  общается с окном через NotificationCenter). Выбор файла — NSOpenPanel;
  прогресс и итоговая сводка — шит поверх окна, импорт можно отменить
  (записи создаются одним батчем в конце — отмена ничего не оставляет).
- **Сетевой этап**: карточка каждого дела открывается прямым GET (капчи на
  карточках нет, троттлинг клиента ~1.5 с/запрос; 380 дел ≈ 10 минут) —
  из карточки берётся УИД.
- **Сшивание по УИД**: в LegalHelp каждая инстанция и каждый материал —
  отдельная карточка; при импорте карточки с одним УИД собираются в ОДНО
  дело. Якорь — низшее звено вида «дело» (район < субъект < АСОЮ < КСОЮ,
  при равенстве — картотека 1-й инстанции); остальные карточки группы
  (вышестоящие инстанции, материалы) — «известные прямые ссылки»
  (`MovementContext.knownCards`). Материал без пары-«дела» — отдельная
  запись. Повторный импорт дублей не создаёт (upsert по ключу записи).
- Все импортированные записи попадают в подборку «Импорт дд.мм.гггг».
  До фоновой загрузки движения в списке видны суд, номер и стороны из
  выгрузки (`essence`); движение подтянет обычный обход RefreshCenter
  (записи без `movementFetchedAt` — первые в очереди), открытие дела
  грузит его сразу.
- Неподдерживаемые платформы пропускаются с отчётом в сводке: мировые
  судьи (msudrf.ru), Мосгорсуд (mos-gorsud.ru), прочие не-sudrf домены.
- Разбор ссылки: звено по домену (справочники КСОЮ/АСОЮ/субъектов, военные
  vap/vkas/ОВС, иначе районное), код субъекта района — по региональному
  суффиксу домена (`syktsud--komi` ↔ `vs--komi` → «11», новый
  `CourtDirectory.subjectCode(forRegionSuffix:)`), картотека — по паре
  (delo_id, new) с фолбэками: ссылки выгрузки бывают неканоническими
  (карточка КСОЮ открыта как `delo_id=2800001`, каноническая пара картотеки —
  `5&new=2800001`).

### Прямые ссылки вместо капчи — `KnownCard`

- `MovementService` принимает список известных прямых ссылок на карточки
  дела. Сквозной поиск по УИД остаётся первичным (он находит все круги), но:
  - **капча** на форме вышестоящего суда больше не даёт заглушку, если
    карточка этого суда известна по прямой ссылке — инстанция собирается
    прямым GET (карточки капчой не закрыты);
  - при **пустом УИД** базовой карточки (сквозной поиск невозможен) и для
    доменов вне подсудности known cards подтягиваются добором, без дублей.
- `MovementContext.knownCards: [KnownCard]?` — опциональное поле, старые
  сохранённые контексты декодируются без миграции.

### Материалы в карточке дела

- Новый уровень инстанции `CaseInstance.Level.material`: в карточке дела —
  секция «Материалы» в конце (те же блоки: заседания, результат, текст
  акта; серая палитра). В стадию/шаги дела материалы не входят; их
  заседания попадают в снимок (календарь/«ближайшее») автоматически.
- Источники: known cards импорта + **поиск по УИД в картотеке «m»**
  домашнего районного суда при сборке движения (аналог добора «других
  кругов»; ошибки и капча глушатся молча — материалы не повод для
  заглушки). Цена — ~2 доп. запроса на дело при фоновом обходе.
- Названия актов: «Определение» (13-…, 13а-…, 15-…) / «Постановление»
  (3/…, 4/… — уголовно-процессуальные материалы).

### Тесты (146, все зелёные)

- `CaseImportTests` — CSV-парсер (кавычки, запятые в сторонах, CRLF — в
  Swift «\r\n» это ОДИН Character), классификация реальных ссылок выгрузки
  (район/субъект/КСОЮ/АСОЮ, точечная форма хоста, материалы, пропуски),
  сшивание по УИД (выбор якоря, материалы-одиночки, «холодные» карточки),
  новые хелперы справочника.
- `KnownCardMovementTests` — капча → прямая ссылка вместо заглушки (и
  прежняя заглушка без known card), добор при пустом УИД, материалы по
  УИД в «m», молчаливое глушение капчи m-поиска, отсутствие дублей,
  сортировка и названия актов материалов.
