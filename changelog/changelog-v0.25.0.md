# Изменения — Alpha 0.25.0

## Суды Москвы: поддержка портала mos-gorsud.ru

Все суды Москвы (Мосгорсуд и районные) живут НЕ на платформе ГАС «Правосудие»,
а на едином портале mos-gorsud.ru — раньше приложение пропускало их как
неподдерживаемую платформу. Теперь московские дела ищутся, открываются и
отслеживаются: у портала обычный GET-поиск без капчи (UTF-8), параметры формы
выверены по боевому паттерну tochno-st/sudrfscraper. Ключевая находка оттуда же:
пустой `courtAlias` ищет по ВСЕМ судам города сразу (название суда возвращается
в строке выдачи) — справочник алиасов районных судов не понадобился.

### Endpoint и модели — `Sources/SudrfKit/MosGorSud.swift`

- `MosGorSudEndpoint.searchURL(courtAlias:uid:caseNumber:participant:instance:processType:)`:
  `/search?formType=fullForm&…` — instance 1/2/3 (первая/апелляция/кассация),
  processType: 1 КАС · 2 гражданские · 3 КоАП · 5 материалы · 6 уголовные.
  Все поля запроса можно передать разом (в отличие от sud_delo).
- `MosGorSudResult` (строка выдачи), `MosGorSudCard` (карточка). Тексты актов
  на портале публикуются ВЛОЖЕНИЯМИ (PDF/DOC) — вместо инлайнового текста
  `actLinks: [URL]`.
- `MosGorSudRouting.map(cartoteka:)` — наши картотеки → (processType, instance):
  `u*` → уголовные, `p*` → КАС, `adm*` → КоАП, `m` → материалы, `g*` →
  гражданские; суффикс id — инстанция. `isMosGorSud(domain:)` — детектор портала.

### Клиент — `Sources/SudrfKit/MosGorSudClient.swift`

Actor по образцу VSRFClient: UTF-8, троттлинг 2 с (перед порталом Qrator),
3 попытки с backoff. TLS — тот же `SudrfTLSDelegate` (mos-gorsud.ru уже в его
списке, сертификаты на корнях Минцифры).

### Парсеры — `Sources/SudrfKit/MosGorSudParsers.swift`

Best-effort (живой HTML из песочницы недоступен): опорная точка выдачи —
ссылки на карточку (`/details/`), УИД и даты распознаются по формату; поля
карточки — пары «подпись → значение» (строки таблиц из двух ячеек, dt/dd),
заседания — строки от даты, акты — ссылки на вложения. TODO: снять живые
фикстуры `mgs_search_uid.html` / `mgs_card.html` и добавить тесты на реальной
разметке (синтетические уже есть).

### Движение дела — `Sources/SudrfKit/MosGorSudMovement.swift`, `Movement.swift`

- Протокол `MosGorSudProviding` (подменяется в тестах), `MovementService`
  получил опциональный клиент `mosgorsud`.
- `moscowMovement(for:cartoteka:)`: карточка базовой инстанции (УИД добирается
  из неё) → апелляция и кассация Мосгорсуда по УИД на самом портале
  (instance=2/3) → 2-й КСОЮ на общей платформе sudrf тем же УИД-циклом
  (упрощённый вариант основного: в кассации без классификации кругов) → ВС РФ
  (по УИД). Капча КСОЮ даёт привычную заглушку-инстанцию.
- Общая точка входа `movement(for:)` распознаёт домен портала и сворачивает на
  московскую ветку — перезапрос отслеживаемого дела (RefreshCenter) работает
  без изменений своего кода.
- `CaseInstance` получил `actURL: URL?` (акт-вложение; Codable-опционал —
  старые кэши декодируются без миграции).

### SudrfApp

- `SearchModel`: для региона «Город Москва» в звене районных судов первой
  опцией идёт «Все районные суды Москвы (портал mos-gorsud.ru)»; у Мосгорсуда
  в звене субъектов снята пометка «вне платформы sudrf» (у Н.Новгорода, Пензы,
  Ульяновска осталась). Ветки портала в `runSearch` (все поля разом, без
  капчи) и `openCard` (карточка по ссылке из выдачи; сводка полей + ссылки на
  акты-вложения). Движение — через общий `service.movement`.
- `MovementContext.makeService` и `RefreshCenter` передают клиент портала.
- `CaseMovementView`: чип «акт (файл)» — ссылка на вложение у инстанций
  mos-gorsud.

### Проверка вживую (из песочницы сеть до судов закрыта)

Регион «Город Москва» → «Все районные суды Москвы» → поиск по № дела/УИД/ФИО;
открыть карточку (сводка + ссылки на акты); двойной клик — движение с
апелляцией МГС и кассацией 2-го КСОЮ; взять дело в отслеживание и обновить.
Снятые страницы сохранить фикстурами в Tests/SudrfKitTests/Fixtures — парсеры
подстроить по реальной разметке.
