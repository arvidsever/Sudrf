# Изменения — v34

## Календарь · «Повестка» — переверстана под большое число дел

В режиме **«Повестка»** (4C) раздела «Календарь» при большом числе отслеживаемых
дел вёрстка «разъезжалась»: правая колонка со списком дней росла вниз без потолка
по высоте и обрезалась, а карточка «Ждут подтверждения» слева тоже не имела
внутренней прокрутки — длинный список сроков выдавливал мини-календарь и ломал
колонку. Переверстка выполнена по хэндоффу из Claude Design (вариант **1A**),
внешний вид сохранён.

Правки только в `Sources/SudrfApp/CalendarScreen.swift`.

## Правая колонка — «липкие» дни вместо боковой колонки даты

`agendaList` переписан с боковой колонки-даты (`HStack` + `.frame(width: 150)`) на
`LazyVStack(pinnedViews: [.sectionHeaders])` с `Section(header:)` — нативный аналог
`position: sticky`. Заголовок каждого дня прилипает к верху, пока прокручивается
его группа, и выталкивается следующим днём. `ScrollView` теперь реально
прокручивает список (его высоту ограничивает негибкий по высоте родитель
`agendaMode`, `body` повестку в свой `ScrollView` не оборачивает).

Новый `dayHeader(_:count:)`: заголовок дня (12.5/700, синий если сегодня),
опциональный чип «сегодня», тонкая линия-разделитель и **счётчик событий**
(«N событий», склонение через `DateUtil.plural`). Под заголовком —
маскирующий градиент из `Color(nsColor: .sudrfContent)` (фон окна, #f2f3f6 в
светлой / #232428 в тёмной теме), чтобы карточки не «просвечивали» под
закреплённой шапкой в обеих темах.

`agendaRow(...)` не изменён.

## Карточка «Ждут подтверждения» — прокрутка внутри себя

Список сроков (`ForEach(waiting)`) обёрнут в `ScrollView { LazyVStack }`, а сама
карточка получила `.frame(maxHeight: .infinity)` — заполняет оставшуюся высоту
левой колонки под мини-календарём. Шапка («ЖДУТ ПОДТВЕРЖДЕНИЯ» + чип-счётчик),
описание и блок пустого состояния закреплены над скроллом. Мини-календарь больше
не сжимается и не сдвигается длинным списком.

## Мини-календарь — индикация дня (тип-точки + хитмап)

После переверстки повестки стал заметен старый баг мини-календаря: он рисовал
**по точке на каждое событие** в неотбивающемся ряду — при 10+ делах в день ряд
перерастал ячейку (~21 px) и уползал в соседние колонки. По хэндоффу (вариант
**1C**) `miniCell` переведён на два непереполняющихся сигнала:

- **Тип-точки (≤ 3):** одна точка на *присутствующий тип* события в порядке
  заседание (`#0a7aff`) → расчётный срок (`#e0932a`) → подтверждённый срок
  (`#d6453a`). День с 12 сроками = одна оранжевая точка, точек всегда не больше
  трёх (новый хелпер `miniDots(_:)`).
- **Хитмап:** плитка `RoundedRectangle(cornerRadius: 8)` под числом тонируется
  акцент-синим по количеству событий за день
  (`opacity = min(0.055 + count/15·0.185, 0.25)`, пустые дни — без заливки) —
  «загрузка месяца» видна одним взглядом. Кружок «сегодня» рисуется поверх плитки.

Точки сроков намеренно ярче `Palette.proposed`/`Palette.confirmed`: мелкие 4-px
точки так читаются лучше. Высота ячейки и переход в режим «Месяц» по клику — без
изменений.

## Совместимость

Карточки событий, цвета, кнопки «Подтвердить / Изменить», клик по заседанию
(→ `openCase`) и логика подтверждения сроков — без изменений. Использованы только
существующие хелперы (`DateUtil`, `StatusChip`, `CardBox`, `Palette`), ничего
нового не введено. Форматы кэша, контекста и моделей не затронуты.
