# Что изменилось (v5 — вышестоящие инстанции)

Движение 1-й инстанции уже работало (v4). Здесь — подтягивание апелляции и кассации.
Причины я проверил на живых сайтах по реальному УИД 11RS0001-01-2025-011255-03.

## 1. Апелляция (ВС Коми) — не тот домен

Домены вышестоящих судов были заданы как `vs.komi.sudrf.ru`, а модульные страницы
(`modules.php`) у судов субъектов открываются только через **двойное тире**:
`vs--komi.sudrf.ru` (точечная форма для модулей редиректит на дефисную). Поиск по
УИД на `vs--komi.sudrf.ru` идёт напрямую, без капчи.

Исправлено:
- `SearchModel`: домен ВС Коми → `vs--komi.sudrf.ru`.
- `CourtDirectory`: запись ВС Коми → `vs--komi.sudrf.ru`.
- `Movement.swift`: `courtLevel`/`shortCourtName` понимают форму `vs--<регион>`;
  название инстанции берётся из справочника (`CourtDirectory.court(forDomain:)`).

### Автоперебор форм хоста (по вашему замечанию — у всех судов одинаково)

`SudrfHost` + обёртка `withHostFallback` в `SudrfClient`: модульные запросы
(`search`, `fetchCard`) идут на дефисную форму хоста, а при сетевой/HTTP-ошибке
повторяются на точечной (перебор обоих вариантов). Капча хостом не лечится —
пробрасывается дальше. Бессегментные хосты (`3kas`) не трогаются.

> По этому УИД ВС Коми возвращает **две** апелляции (33-4818/2025 и новую
> 33-2266/2026 — после кассации дело ушло на новое рассмотрение). Сейчас берётся
> первая найденная. Если нужно показывать оба круга — отдельная доработка.

## 2. Кассация (3 КСОЮ) — форма под капчей

Форма поиска 3 КСОЮ защищена кодом с картинки (поле «Проверочный код», картинка —
inline `data:`-изображение). Автопоиск по УИД невозможен. При этом **карточка**
кассации (`name_op=case`) капчей не защищена и грузится напрямую — проверено.
Перекрёстных ссылок между инстанциями в карточках нет (только текст), так что
обойти поиск нельзя.

Сделано:
- `CaptchaDetector`: добавлено распознавание этой капчи по тексту «проверочный код» /
  «код с картинки» (старые селекторы её не ловили — картинка отдаётся как `data:`-URI).
  Проверено, что форма ВС Коми этих слов не содержит → апелляция по-прежнему грузится
  автоматически (ложного срабатывания нет).
- `MovementService`: при капче вышестоящего суда вместо «тихо пропустить» добавляется
  **инстанция-заглушка** с URL формы (`CaseInstance.captchaFormURL`).
- Окно ввода кода (`CaptchaWebView` / `CaptchaSheet`, WKWebView): открывает настоящую
  форму суда (УИД подставляется автоматически), **код вводит пользователь** —
  приложение капчу не решает. Когда пользователь доходит до карточки дела,
  приложение считывает её HTML и через `CaseCardParser` заполняет инстанцию.
- `CaseMovementView`: на заглушке — кнопка «Ввести код».
- `SearchModel`: состояние `captcha`, методы `beginCaptcha(_:)` и `ingestCaptchaCard(_:)`.
