# Изменения — Alpha 0.30.0

## Уточнение раскладки участников КоАП/УПК

Донастройка распознавания участников из v29 по замечаниям пользователя.

### 1. КоАП: «Представитель» → сторона обвинения

В КоАП роль «Представитель» (без уточнения) — это обычно представитель
потерпевшего, поэтому теперь он попадает на **сторону обвинения**, а не защиты.
Защитник по-прежнему на стороне защиты, привлекаемое лицо — на защите.

### 2. УПК: «Представитель учреждения (компетентного органа)» → «Иные лица»

Добавлена **третья колонка «Иные лица»** (значок — как у третьих/заинтересованных
лиц ГПК/КАС, `PartyIcon.person`). Роль «Представитель учреждения (компетентного
органа)» больше не приписывается ни к защите, ни к обвинению.

Раскладка по сторонам вынесена в отдельную функцию `CaseParties.side(forRole:kind:)`
с явным порядком проверок (учреждение → обвинение → защита → голый представитель →
по умолчанию защита), `buildSided` теперь строит до трёх колонок.

### 3. Перечень статей: карточка vs «Списком»

Перечень статей подсудимого/привлекаемого лица вынесен из под-роли в отдельное
поле `PartyMember.articles` (и `RoleItem.articles`). Парсер собирает статьи из
обеих раскладок: у УПК — из вкладки «ЛИЦА» (`parsePersonsTable`), у КоАП — из
колонки «Перечень статей» таблицы «СТОРОНЫ ПО ДЕЛУ» (`parsePartiesTable`).

Отображается по-разному в двух местах:

- **Карточка дела** (`CaseMovementView`) — под-роль со словом-ролью и статьями:
  «Подсудимый · ст.173.1 ч.2 п.б УК РФ», «ПРИВЛЕКАЕМОЕ ЛИЦО · …». Без щита.
- **Таблица «Списком»** («Мои дела», `MyCasesView`) — ФИО, значок **щита** (тот же
  `shield`, что у ответчиков ГПК), затем перечень статей, **без слова**
  «Подсудимый»/«Привлекаемый». Статьи прокинуты структурно через
  `CaseParties.leadCharges` → `CaseSnapshot.leadCharges` → `TrackedCase.leadCharges`
  (SF Symbol нельзя положить в хранимую строку `partiesShort`).

## Совместимость

Новые поля `articles` — опциональные, старый кэш движений (`MovementCodable`)
декодируется без изменений (отсутствующий ключ → `nil`).
