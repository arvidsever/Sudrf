# Изменения — Alpha 0.20.0

## Редизайн раздела «Мои дела» (хэндофф Claude Design)

Интеграция дизайн-пакета `design_handoff_liquid_glass 3`: новый состав и
порядок видов, «подборки» вместо «папок», вид производства с палитрой-шкалой
тяжести, двухэтажная таблица без обрезаний, три секции фильтров, живой поиск,
сортировка и drag-and-drop. Референсы писались от v15 — правки перенесены в
живую базу v19 с сохранением её механизмов (открытие дела по `recordKey`,
шапка с TTL-меню и RefreshCenter, персистентные снимки SwiftData).

### Виды раздела

Порядок фиксированный, дефолт — «Списком»:

1. **Списком** — панель-фильтр слева + таблица (по умолчанию);
2. **По стадиям** — карточки, группы-стадии;
3. **По производствам** — карточки, группы по виду производства (новый вид);
4. **По подборкам** — карточки, группы-подборки (бывш. «По доверителям»).

### Подборки вместо папок

- Подборка — доверитель, тема, что угодно; **одно дело может состоять в
  нескольких подборках** (`TrackedCaseRecord.collectionNames: [String]`,
  лёгкая миграция SwiftData; legacy-`folderName` один раз пересаживается в
  подборки и очищается).
- Пустые подборки переживают перезапуск: список созданных имён — в
  UserDefaults (`myCollections`); имена, встреченные на делах, впитываются.
- Членство — два способа: **значок папки** в строке таблицы (меню-чеклист)
  и **drag-and-drop** строки на пункт подборки в сайдбаре (подсветка цели
  fill accent 0.2 + ring 0.55; «Все дела» drop не принимает). Payload —
  `recordKey`, не номер дела (номер без суда неоднозначен).
- **«+ Новая подборка»** — инлайн-поле: Enter создаёт и выбирает, Esc
  отменяет; пустое/дублирующее имя игнорируется.

### Вид производства (`ProductionType`)

- Определяется по префиксу номера: `1-` уголовное · `5-` дело об АП ·
  `2а-/3а-/8а-/33а-` административное (КАС) · остальные — гражданское
  (регистронезависимо — на портале встречается заглавная «А»).
- Палитра «шкала тяжести» — цвет кодирует серьёзность, а не «тип»:
  гражданское `#607890` · КАС `#a3782a` · КоАП `#c05c2a` · уголовное `#9e2b2b`.
- Применение: буква-бейдж `Г/А/У/АП` в сайдбаре, подпись вида под номером
  в таблице, постоянная подкраска строк цветом вида (`color.opacity(0.06)`;
  в хэндоффе была «за флагом, выкл по умолчанию» — решено включить всегда).

### Таблица «Списком»

- Было 7 узких колонок с эллипсисами — стало **5, три из них двухэтажные,
  ничего не обрезается** (`fixedSize`): точка «непросмотренное» · Дело · вид ·
  Стороны · суд · Статус · событие · Дальше (+ значок подборок).
- Разделитель сторон — **скрещенные мечи «⚔»** вместо стрелки
  (`MovementDerivation.partiesShort`; старые снимки мигрируются при
  отображении заменой « → » на « ⚔ »).
- **Сортировка** (меню у заголовка): по активности (дефолт, свежие изменения
  сверху — `lastEventDate`), по ближайшему событию (`nextEventDate` — минимум
  из будущих заседаний и сроков), по номеру дела (numeric compare).
- **Живой поиск** в сайдбаре: вхождение в номер + стороны + подборки + суд,
  case-insensitive, ✕ очищает; пустой результат — центрированная заглушка.

### Сайдбар вида «Списком»

Три секции сверху вниз: **ПОДБОРКИ** (с живым поиском) → **ВИД
ПРОИЗВОДСТВА** → **СТАДИЯ**. Фильтры комбинируются (И); повторный клик по
активному пункту производства/стадии снимает фильтр.

### Исправление: строки не тянулись на всю ширину

Строки и шапка таблицы были собраны на `LazyVGrid` с `flexible(min, max)`-
колонками, а грид не отдаёт остаток ширины от колонок, упёршихся в свой max,
безлимитной колонке «Стороны · суд» — в широком окне строки занимали ~60%
карточки. Заменено на `HStack` с min/max-фреймами ячеек (те же дизайн-токены
`12 · 86–150 · ≥152 flex · 128–210 · 94–130`): HStack остаток докладывает,
«Дальше» со значком папки прижимается к правому краю.

### Исправление: вертикальная полоса в таблице

Рецидив бага из v18: `.overlay(Divider(), alignment: .top)` на label внутри
`Button` раскладывает `Divider` вертикально — полоса на всю высоту по центру
ширины. Разделитель строки таблицы перенесён с label на сам `Button`
(как в v18 для `OverviewView.hearingsColumn`).
