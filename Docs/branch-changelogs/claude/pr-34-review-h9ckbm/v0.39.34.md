# v0.39.34 — корректные КоАП-цепочки импорта и ремонта

- Процессуальная роль КоАП-карточки теперь определяется по звену суда,
  картотеке и происхождению УИД: `MS` обозначает мирового судью, `RS` —
  районный суд. Районный `admj + MS` считается апелляцией, а `admj + RS` —
  первым судебным пересмотром постановления несудебного органа.
- Маршруты трёх картотек суда субъекта разделены явно: районный `adm` ведёт в
  `adm1`, районный `admj + RS` — в `adm2`, а вступившие в силу акты — в
  `adm33` суда субъекта либо `adm3` КСОЮ по применимой переходной дате.
  `admj + MS` никогда не направляется в `adm2`.
- Учтены переходы 01.10.2019 и 10.05.2026. При неизвестной дате безопасно
  проверяются оба допустимых портала; с 10.05.2026 MS-кассация ищется только
  в суде субъекта.
- Импорт выбирает якорь после загрузки карточки и настоящего УИД. Для
  `MS-admj` восстанавливается точная карточка мирового судьи `adm`; отсутствие
  промежуточной районной жалобы допустимо, а список районных судов сохраняется
  для последующего поиска необязательной апелляции.
- Автоматический ремонт обновлён до `importChainRepair.v2`: повторно
  обрабатывает старые неподдержанные записи, исправляет уровни и точные цели
  КоАП, переякоривает только `MS-admj`, сохраняет `RS-admj` каноническим и
  отдельно сообщает число исправленных маршрутов.
- Добавлены регрессии для MS/RS, всех трёх областных картотек, исторического
  `adm33`, перехода 10.05.2026, импорта, ремонта и повторного идемпотентного
  запуска.
