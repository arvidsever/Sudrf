### v0.38.0 — SwiftPM `CaptchaSolver`

Новый SwiftPM-продукт `CaptchaSolver` (Sources/CaptchaSolver/) —
Vision-based, on-device, no network. По умолчанию включён;
отключается в системном меню «Captcha» (⌃⌘A). Подключён в
`RefreshCenter.tryAutoSolve` для фонового обхода; manual sheet
становится fallback. Три call site подготовлены.

### v0.38.1 — Wire auto-solver в интерактивный `SearchModel`

`AutoCaptchaSolver.solve` подключён к `SearchModel.runSearch`.
`executeSearch(allowAutoSolve:)` + `handleCaptcha(...)` flow.

### v0.38.2 — Wire auto-solve в per-instance captcha stub

`AppRouter.beginCaptcha(for:)` теперь использует solver. Третий
кассационный путь (`CaseMovementView`).

### v0.38.3 — Log fix + failed-image dump

`CaptchaSolverLog` использует `FileHandle(forUpdating:)` +
`seekToEnd()` — `forWritingTo` overwrote from offset 0.
`logFailedImage(...)` пишет PNG в
`~/Library/Application Support/Sudrf/captcha-failures/`. FIFO 50
файлов.

### v0.38.4 — Per-host preprocessor

`Preprocessor` (grayscale + contrast + 2× scale), per-host gated.
Default OFF — регрессирует на простых captcha. `preprocessorHosts:
Set<String>` для opt-in. `minimumTextHeight` 0.3 → 0.2.

### v0.38.5 — SearchDiagnostics (HTML dumper)

Пишет в `~/Library/Application Support/Sudrf/diagnostics/`. Три
пути: `dumpFormCheck`, `dumpVariant`, `dumpSolverMismatch`.
Toggle: `defaults write ru.sudrf.app captcha.diagnosticsEnabled -bool NO`.

### v0.38.6 — Raw bytes в дампе

Diagnostic dump сохраняет **raw bytes** (без перекодирования в
UTF-8). v0.38.5 писал `String.data(using: .utf8)` → mojibake при
`Content-Type: text/html; charset=windows-1251`. v0.38.6 — новые
`data:`-перегрузки + `SudrfClient.fetchHTMLData`.

### v0.38.7 — Live preprocess toggle + top-3 candidates

`CaptchaSettings.preprocessorEnabled` стал live-флагом через
`VisionOCRStrategy.preprocessingProvider: (() -> Bool)?` — тоггл
в меню «Captcha» действует сразу. `AutoCaptchaSolver` пишет
per-attempt diagnostic файл
`~/Library/Application Support/Sudrf/diagnostics/<host>_<ts>_<kind>_candidates.txt`
с топ-3 кандидатами Vision и пометкой `preprocessed=yes/no`. Real-PNG
тесты на основе `captcha-failures/` с XCTSkip-when-empty.

## Контекст

К v0.38.7 (тоггл preprocess в меню + top-3 candidates diagnostic)
симптоматическая диагностика закрыта: при отказе солвера у нас
есть PNG, candidates.txt и per-host log. Дальше — собственно
**исправление точности**: на `1kas`, `oblsud--mo.sudrf.ru`,
`sankt-peterburgsky--spb.sudrf.ru` солвер выдаёт неверный код.
По новому свидетельству (1kas variant diag: «Неверно указан
провочный код с картинки») — это **OCR-проблема**, не URL/cookie/
headers. Vision слаб на rotated/struck-through captcha.

v0.38.8 заменяет/дополняет Vision **CoreML-моделью**, обученной
на размеченном корпусе (4042 captcha от друга). Это инвестиция
в on-device OCR, не уход в сетевые антикапча-сервисы.

## Архитектура (по описанию друга)

Вход: 100×30 RGB → бинарная маска «чернил» (порог по цвету
~(2, 103, 154)) → downsample 100×30 → 64×20 (box averaging).
Выход: `[1, 5, 10]` — 5 softmax-голов по одной на позицию цифры.

```
input 64×20×1 binary ink mask
  → conv 3×3, 8 filters, leakyReLU, maxpool 2×2
  → conv 3×3, 16 filters, leakyReLU, maxpool 2×2
  → dense 64, leakyReLU
  → 5 × softmax(10)  (5 позиций × 10 цифр)
```

Обучение: mini-batch SGD (24), lr=0.02, ×0.5 на эпохах 10/16/22,
momentum 0.9, L2=1e-4, ранняя остановка. Friend reports 95.1%
per-digit на 4039 captchas.

  `CaptchaSolvingProvider`, прогоняет PNG через ту же бинаризацию

## Release notes (lift to `changelog/changelog-v0.38.8.md` at merge)

- Обучена CoreML-модель `model-captcha-numeric.mlmodelc` на 4042
  размеченных captchas (друг).
- Добавлен `CoreMLCaptchaStrategy(kind: .sudrfToken)`, реализующий

  conv×2 → dense → 5×softmax(10).

  `AppModel` и `SearchModel` загружают модель из
  `Bundle.main` или `~/Library/Application Support/Sudrf/`,
  fallback на `VisionOCRStrategy` когда отсутствует.
- +4 теста (CoreMLCaptchaStrategyTests), всего 279.
- **BREAKING для пользователей captcha-включённых судов:**
  точность солвера на `1kas`, `oblsud--mo.sudrf.ru`,
  `sankt-peterburgsky--spb.sudrf.ru` ожидаемо вырастает с ~0%
  (Vision conf=0.00) до ~85-95% (CoreML).

## Как тестировать вживую

1. `bash Scripts/train-coreml-captcha.swift --output Tests/CaptchaSolverTests/Fixtures/model-captcha-numeric.mlmodelc/`
2. `swift test` — все 279 тестов зелёные.
3. `swift run SudrfApp`, попробовать поиск на 1kas, oblsud--mo,
   sankt-peterburgsky--spb. Ожидаем, что captcha больше не
   появляется (или появляется редко).
4. Проверить `~/Library/Application Support/Sudrf/captchas.log`
   (или `captcha-solve.log`) — там должны быть попытки с
   conf > 0.85 (CoreML) вместо conf=0.00 (Vision).

## Что не входит в v0.38.8

- **Kcaptcha CoreML** — отдельная модель, отдельный корпус,
  отдельная задача. Запланирована на v0.39.0 (или позже,
  когда накопится 500+ text-captcha).
- **Self-improving bootstrap** (auto-добавление успешно решённых
  captcha в корпус + retrain) — отдельная инициатива, планируется
  на v0.38.9.
- **Per-host threshold calibration** — после v0.38.8 мы сможем
  собрать per-host accuracy stats и подобрать пороги.
