# Изменения — v26

## Доводка VNKOD-судов по живым тестам + капча поиска во встроенном окне

Живые проверки v23–v25 выявили три бага: воронежские суды падали с
«NSURLErrorDomain -999 (отменено)», амурские отправляли капчу в Safari вместо
встроенного окна, ульяновские находили дело, но не открывали карточку и не
собирали движение («У записи нет идентификаторов карточки»).

## 1. Винтажная карточка: идентификаторы, URL, парсер (Ульяновск)

Диагноз по живой карточке Заволжского районного суда г. Ульяновска (webarchive
пользователя → фикстура `Tests/SudrfKitTests/Fixtures/zavolgskiy_card.html`):
винтажный интерфейс открывает карточку параметрами
`name_op=case&_id=…&_uid=…&_deloId=…&_caseType=0&_new=…&srv_num=1`
(не `case_id`/`case_uid`/`delo_id`), а разметка карточки другая: шапка
`.case-num` («ДЕЛО № …»), вкладки `tab_content_Case` (метка→значение),
`ClaimList` (требования), `EventList` (движение), `PersonList` (лица) вместо
современных `cont{N}`; у таблиц есть мобильные дубли `.block-mobile`.

- `Sources/SudrfKit/ResultsParser.swift`: `caseID`/`caseUID` извлекаются и из
  `_id`/`_uid` (фолбэк к `case_id`/`case_uid`).
- `Sources/SudrfKit/SudrfURLBuilder.swift`, `cardURL`: ветка для винтажных судов
  в точном формате живого URL; пары `_deloId`/`_new` — через тот же маппинг,
  что у выдачи (`vnkodDeloParams`: апелляция `delo_id=4/new=4` → `_deloId=1540006&_new=4`).
- `Sources/SudrfKit/CaseCardParser.swift`: винтажная ветка (детект по
  `#case_bookmarks`/`tab_content_*`): УИД/судья/категория/дата поступления из
  вкладки «Дело», сессии из «Движение дела» (колонки по шапке thead), стороны
  из «Лица» (существующий сборщик ролей), результат из «Требования» (колонка
  «Решение»); мобильные дубли отсекаются (только таблицы `.non-list`).
  Актов в фикстуре нет — распознавание винтажной вкладки актов отложено
  (TODO: нужна карточка с опубликованным актом).
- Кодировка: винтажный сервер отдаёт UTF-8 (meta врёт про windows-1251) —
  `fetchHTML` уже обрабатывает по Content-Type, изменений не потребовалось.

## 2. Капча базового поиска — встроенное окно + авто-перезапуск (Амурск)

Раньше капча в поиске показывала жёлтый баннер со ссылкой в Safari, а
встроенное окно (`CaptchaSheet`) работало только для заглушек вышестоящих
инстанций. Теперь:

- `SearchModel`: `CaptchaContext` получил `caseNumber` (автоподстановка) и
  `rerunSearch`; catch `captchaRequired` в `runSearch` открывает встроенный
  лист. После решения кода пара captcha/captchaid перехватывается (механика
  v24), лист закрывается и `runSearch` перезапускается сам — пара уходит в URL,
  поиск проходит без повторного окна. Если пользователь дошёл в окне до
  карточки — тот же перезапуск через `ingestCaptchaCard`.
- `ContentView`: лист капчи перевешен с вью движения на всю панель (иначе в
  состоянии поиска он не показывался); `captchaURL` и `CaptchaBanner` удалены.
- `CaptchaWebView`: автоподстановка стала регистронезависимой и заполняет оба
  поля — УИД и № дела. Раньше матчился только ВЕРХНИЙ регистр
  `JUDICIAL_UIDSS`, а винтажные поля (`case__judicial_uidss`) в нижнем.

## 3. TLS: мягкая проверка для доменов судов (Воронеж)

-999 «отменено» — это наш `SudrfTLSDelegate` отменял challenge, когда цепочка
сертификата суда не проходила `SecTrustEvaluateWithError` даже с якорями
Минцифры (у части винтажных судов она кривая). Теперь для доменов судов
(`sudrf.ru`, `mos-gorsud.ru`) сертификат принимается и при провале оценки —
так же ведёт себя встроенное капча-окно, а апстрим sudrfscraper вообще ходит
по http. Для всех прочих хостов проверка строгая, как раньше. Если вживую
останется -999 — следующий шаг: фолбэк на http с ATS-исключением.

## Проверка вживую (из песочницы сеть до судов закрыта)

- Воронеж (Коминтерновский районный): поиск без -999 (выдача, пустота или капча).
- Амурск (Благовещенский городской): капча во ВСТРОЕННОМ окне; УИД/№ дела
  подставлены; после ввода кода окно закрывается и поиск продолжается сам;
  повторный поиск по тому же суду — без окна.
- Ульяновск (Заволжский районный): клик по строке выдачи открывает карточку,
  двойной клик собирает движение (сессии, стороны, УИД 73RS0004-…).
