# Изменения — v35

## CAPTCHA Assist: один быстрый ручной ввод на суд

Добавлен режим **CAPTCHA Assist** для ситуаций, когда сайт суда вместо результатов
возвращает форму с кодом с картинки. Приложение по-прежнему не распознаёт и не
подставляет код автоматически: пользователь сам читает пятизначный код и отправляет
форму в `WKWebView`. Новая логика убирает лишнюю механику вокруг этого шага.

## Удобство ввода

Старый лист с веб-формой заменён на `CaptchaAssistSheet`
(`Sources/SudrfApp/CaptchaWebView.swift`). Встроенный `WKWebView` теперь
автоматически заполняет только обычные поля поиска: УИД и номер дела. Поле CAPTCHA
получает фокус, блок с картинкой прокручивается в видимую область, а сама картинка
увеличивается на странице. Справа добавлена панель с крупным предпросмотром картинки,
индикатором состояния, режимом повышенного контраста и списком дел, которые ждут
этот же код.

JavaScript используется только как инструмент помощи интерфейсу: найти картинку,
сфокусировать поле, прокрутить и увеличить блок. Vision/Core ML/OCR и автоматическая
подстановка текста CAPTCHA не добавлялись.

## Очередь ожидающих дел по суду

`RefreshCenter` теперь группирует фоновые обновления, упёршиеся в CAPTCHA, по
нормализованному хосту суда. Если несколько дел одного суда требуют код, пользователь
видит это в панели CAPTCHA Assist и вводит код один раз для текущей сессии.

После успешной отправки формы приложение сохраняет существующую пару
`captcha/captchaid` через `CaptchaTokenStore`, как и раньше, а затем автоматически
перезапускает все ожидающие обновления для этого же суда. Успешное обновление
удаляет дело из очереди; отклонённый токен по-прежнему инвалидируется и снова
показывает ручной ввод.

## Тесты

Добавлены unit-тесты `CaptchaPendingQueueTests` для группировки dot/dash-вариантов
доменов СУДРФ, очистки очереди после retry и переноса дела между хостами. Полный
прогон `swift test --quiet`: 211 тестов, 0 падений.
