## Контекст

После v0.38.8 у нас есть CoreML-стратегия (требует скомпилированной
модели), но сам корпус **растёт медленно** — только если кто-то
положит PNG в `~/Library/.../captcha-training/solved/`. v0.38.9
замыкает цикл: каждый успешный auto-solve солвера добавляет PNG в
корпус. При превышении потолка — FIFO-eviction, корпус не
растёт бесконечно.

Это основа для self-improving солвера: модель на v0.38.8 (обученная
на 4042 captcha) работает → добавляет новые captcha в корпус →
после +500 captchas пользователю предлагается retrain →
точность растёт дальше.

## Защиты от ложных срабатываний (v0.38.9)

Bootstrap-хуk добавляет PNG в корпус ТОЛЬКО если:

1. **`lastSubmittedCaptchaPNG != nil`** — captcha была недавно
   отправлена (иначе нечего добавлять).
2. **`results.count > 0`** — search вернул ≥ 1 результата
   (real results page, не captcha-rejection). Empty results
   пропускаются, чтобы не плодить шум.
3. **`CaptchaTokenStore.token(forDomain:)` существует** — токен
   не протух, иначе ground truth теряется.
4. **AppModel и RefreshCenter НЕ добавляют в корпус** — там
   нет гарантии, что retry с токеном прошёл. Bootstrap живёт
   только в `SearchModel.executeSearch`, где есть явный сигнал
   «search вернул ≥ 1 результата».

## Release notes (lift to `changelog/changelog-v0.38.9.md` at merge)

- `CorpusStore` (actor) — dual-corpus bootstrap. При каждом
  успешном search с auto-solve PNG captcha добавляется в
  `solved-<kind>/<code>_<host>_<ts>_<uuid>.png`. Потолок 5000
  на kind (numeric + text), FIFO-eviction по mtime.
- `SearchPageKind.captchaRejected` — различает «форму с
  captcha-картинкой» и «отказ после submit'а». Маркеры
  собраны из handoff v0.38.7 (1kas variant diag) и
  распространённых формулировок.
- `AutoCaptchaSolver.solve(...)` теперь возвращает
  `SolveResult { token, png }` — нужно для bootstrap.
- `SearchModel.executeSearch` — bootstrap-хуk: после успешного
  search с auto-solve PNG добавляется в `CorpusStore`.
- `SudrfClient.runVariants` — обрабатывает `.captchaRejected`
  как путь к `searchModuleUnavailable` (с диагностическим
  дампом).
- +9 тестов. Всего 289.
- **Эффект на пользователя:** корпус captcha-изображений
  начинает расти с первого запуска. Через несколько недель
  активного использования у нас будет собственный
  per-host-labeled датасет для переобучения. После
  v0.39.0 (Python-скрипт + retrain UI) — точность солвера
  начинает адаптироваться к реальным стилям капчи на
  пользовательских судах.
