# Изменения — Alpha 0.37.0

## Добавлено

### Мировые судьи

В ветку «Общие» добавлено звено «Мировые судьи». Список судебных участков
подтягивается вживую из справочника ГАС «Правосудие» `go_ms_search` по выбранному
региону. Участки на `*.msudrf.ru` доступны для поиска; московские, петербургские
и другие внешние порталы остаются в списке, но помечаются как неподключённые.

Для `msudrf.ru` добавлены отдельные URL-builder, клиент, парсер выдачи и парсер
карточки. Поддержаны поиск по номеру дела и ФИО, пагинация `pageNum_Recordset1`,
дедупликация строк, разбор движения, сторон, УИД, даты вступления в силу и текста
судебного акта. Поиск по УИД для мировых отключён, потому что в форме участка
такого поля нет.

### CAPTCHA для участков

Для мировых участков добавлен отдельный поток KCaptcha: буквенно-цифровой код,
форма `kcaptchaForm`, поле `captcha-response`, картинка `/captcha.php` и перенос
cookie/PHPSESSID из скрытого `WKWebView` в общий cookie store. После успешного
ввода приложение повторяет ожидающий поиск или обновление движения.

### Движение дела

Для дел мировых судей движение строится от базовой карточки: районная апелляция
ищется по УИД во всех районных судах региона, первая кассация — до 01.01.2026 в
КСОЮ, с 01.01.2026 в президиуме суда субъекта, а при отсутствии даты вступления
в силу — по обоим направлениям. Прямой кассационный поиск от акта мирового судьи
сохраняется. Поиск в ВС РФ подключён тем же механизмом, что и для остальных дел.

## Изменено

В сетевом слое добавлен доверенный домен `msudrf.ru`, ATS-исключение для
приложения и HTTP-fallback только для `*.msudrf.ru`, если HTTPS падает именно на
TLS. Для `msudrf.ru` минимум TLS в Info.plist оставлен на `TLSv1.2`, чтобы не
создавать лишнее предупреждение Xcode; старое исключение `sudrf.ru` не менялось.

## Исправлено

Исправлен крэш `Swift/ContiguousArrayBuffer.swift: Fatal error: Index out of
range` в списке результатов. Строки выдачи теперь имеют стабильный `stableID`, а
выбранная карточка хранится по ID, а не по индексу массива. Если результаты
обновились, старый tap/double-tap тихо игнорируется вместо обращения к уже
несуществующему индексу.

Также устранено предупреждение SwiftUI `Modifying state during view update` в
CAPTCHA Assist: callbacks из `WKWebView` больше не меняют `@State` синхронно из
пути `updateNSView`.

## Тесты

Добавлены регрессионные тесты для справочника мировых, парсинга выдачи и карточки,
KCaptcha, правил кассации до/после 01.01.2026, а также тесты стабильного выбора
строки выдачи по `stableID`.

Полный прогон `swift test --no-parallel`: 227 тестов, 0 падений.
