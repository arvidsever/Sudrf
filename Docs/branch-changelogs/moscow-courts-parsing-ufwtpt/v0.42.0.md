# v0.42.0 — Мосгорсуд: парсинг выверен по живому порталу

Черновик release-нот ветки `claude/moscow-courts-parsing-ufwtpt`. Номер —
прогноз, финал присваивается при merge (см. `AGENTS.md`).

## Суды Москвы (mos-gorsud.ru) — парсинг приведён к боевой разметке

Прежняя реализация Мосгорсуда была написана по угаданной вёрстке и на живом
портале не работала (тесты проходили только на синтетике). Сверено с рабочим
парсером коллеги и **живыми фикстурами портала** (карточки + страница выдачи).

### Исправлено
- **Выдача поиска.** Строки результата — это `<tr data-href="…/services/cases/
  <section>/details/<uuid>">`, а не `<a href>`. Парсер переписан: строки берутся
  по `data-href`, колонки читаются по заголовкам `<thead>` (№ дела, стороны,
  состояние, категория, судья), раздел — из сегмента пути.
- **Карточка дела.** Поля лежат в парах `<div class="left">КЛЮЧ</div>
  <div class="right">ЗНАЧЕНИЕ</div>` (а не `<th>/<td>`); «Судья» на портале
  пишется латинской «C» («Cудья») — ключи ищутся по вхождению; заседания и
  состояние — таблицы по заголовкам («Зал»/«Состояние»); тексты актов —
  вложения по ссылке `/…/cases/docs/content/<uuid>` (а не `attach/.pdf`).
- **Коды инстанции.** Кассация на портале — `instance=4` («Кассационная»);
  `3` — это «Второй пересмотр» (надзор). Раньше кассация ошибочно кодировалась
  как `3` (в `MosGorSudRouting.map` и в московской ветке движения) — исправлено.
- **URL поиска** приведён к живой форме портала: `courtAlias` + одно из
  `uid/caseNumber/participant` + `instance` + `processType`; лишние `page`/
  `formType` убраны.

### Добавлено
- **Дизамбигуация раздела.** Короткие номера пересекаются между видами
  производства/инстанциями (уголовные `01-…` vs апелляция `10-…` vs КоАП
  `05-…`/`12-…`). Введён детерминированный фильтр строк выдачи по сегменту пути,
  построенный из официальной модели портала (`instanceTypes`/`processTypes` ×
  `mgsLinksMapping`/`rsLinksMapping` из его `scripts.js`): выбранные
  (вид, инстанция) → пересечение ключей → допустимые сегменты (`first-civil`,
  `appeal-criminal`, `admin`, `appeal-admin`, …). Коды видов производства
  (КАС=1, гражд.=2, КоАП=3, материалы=5, уголов.=6) сверены и верны.
- **Тесты на живых фикстурах** (`Tests/SudrfKitTests/Fixtures/mosgorsud/`):
  карточка гражданского дела, карточка КАС и страница выдачи — закрывают прежний
  TODO про «снять живые фикстуры».

### Известные ограничения
- На портале подключена Yandex SmartCaptcha; при умеренной нагрузке она не
  всплывает (обхода нет, как и у исходного парсера). Если начнёт появляться —
  отдельная задача.
- Мировые судьи Москвы (`mos-sud.ru`) — отдельный портал, в этот заход не
  подключались.
