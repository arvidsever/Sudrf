# 0.40.3 — Стабилизация флаки-теста троттла VSRF

> Alpha, build 83.

## Исправлено

- `testConcurrentRequestsReserveSeparateThrottleSlots` больше не флакует на CI.
  Тест меряет моменты `startLoading` в URLProtocol-стабе (слот троттла +
  джиттер доставки), поэтому жёсткий порог 0.11с на отдельный интервал падал
  при исправном троттле (наблюдалось 0.091с на main, run 96). Пороги
  привязаны к `minInterval`: floor `minInterval / 3` на каждый интервал
  отделяет регрессию «резервация после await» (интервалы ~0–5мс) от шума
  планировщика, а новый ассерт на суммарный разбег (`≥ minInterval`)
  устойчив к джиттеру за счёт взаимной компенсации соседних лагов.
- Продакшн-код `VSRFClient` не менялся: слоты троттла резервируются атомарно
  внутри actor до `await`, реализация корректна.

  включая `VSRFClientThrottleTests`.
